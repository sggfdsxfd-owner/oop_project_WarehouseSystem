# 智慧物流與倉庫管理系統 - 系統規格說明書 (System Specification)

本規格說明書詳細定義了「智慧物流與倉庫管理系統」的核心功能、非功能性需求、系統架構、類別與資料結構設計，以及資料持久化格式。

---

## 1. 系統概述 (System Overview)

本系統為一個多使用者、角色型權限控管的倉庫管理系統 (WMS)，採用終端機文字介面 (TUI) 提供直覺、友善的操作。系統核心目的在於模擬物流中心如何妥善儲存不同特性的貨物（包含普通貨物、危險品、易腐品與易碎品），並利用動態綁定機制，根據貨物類型精確計算倉儲費用。

---

## 2. 角色與權限規範 (Roles & Access Control)

系統支援兩種主要角色，並在登入後提供相異的控制台介面與執行權限：

| 功能項目 | 系統管理員 (Admin) | 一般作業員 (Regular) | 說明 |
| :--- | :---: | :---: | :--- |
| **登入/登出** | V | V | 使用者必須通過密碼驗證方能存取系統 |
| **查看所有庫存** | V | - | 管理員可查看倉庫中所有貨主的物品 |
| **查看個人庫存** | - | V | 作業員僅能查看及檢索自己申報/擁有的物品 |
| **申報貨物入庫** | V | V | 管理員可指派任意擁有者，作業員自動設為自己 |
| **辦理貨物出庫** | V | 僅限自己貨物 | 安全性管制，防止未經授權提領他人貨物 |
| **營運統計與報表**| V | - | 統計總容量、營運預估收益與危險品警報 |
| **管理使用者帳號**| V | - | 新增系統使用者帳號 |

## 3. 貨物類別與倉儲費用計算規格 (Polymorphic Billing Specification)

貨物以 `Cargo` 作為基底類別，透過虛擬函式 `calculateStorageFee()` 與 `printDetails()` 實作多型，以下為費率計算與屬性規格：

### 3.1 普通貨物 (`Cargo`)
*   **專屬屬性**：無。
*   **倉儲費用公式**：
    $$\text{費用} = \text{體積 (m³)} \times \text{基本費率 (元/m³)}$$
*   **用途描述**：適用於無特殊存放限制的一般箱裝或棧板貨物。

### 3.2 危險品 (`DangerousCargo`)
*   **專屬屬性**：
    *   `hazardLevel` (整數，範圍 1-9)：危險等級，等級越高安全加成越高。
    *   `unNumber` (字串，如 `UN3480`)：聯合國危險物編號。
*   **倉儲費用公式**：
    $$\text{費用} = (\text{體積} \times \text{基本費率} \times (1.0 + 0.2 \times \text{危險等級})) + 500 \text{ 元}$$
    *(註：$500$ 元為固定收取的每日防爆監控與安全保險基本費)*
*   **警報觸發**：若 `hazardLevel >= 7`，管理員控制台會自動亮起紅色高亮安全警告。

### 3.3 易腐品/冷鏈貨物 (`PerishableCargo`)
*   **專屬屬性**：
    *   `expiryDate` (字串，格式 `YYYY-MM-DD`)：貨物有效過期日。
    *   `requiredTemp` (雙精度浮點數，單位 `°C`)：指定溫控需求。
*   **倉儲費用公式**：
    *   **冷凍級** ($T < 0^{\circ}\text{C}$)：
        $$\text{費用} = (\text{體積} \times \text{基本費率}) + (\text{體積} \times 300)$$
    *   **冷藏級** ($0^{\circ}\text{C} \le T \le 8^{\circ}\text{C}$)：
        $$\text{費用} = (\text{體積} \times \text{基本費率}) + (\text{體積} \times 150)$$
    *   **常溫/無溫控** ($T > 8^{\circ}\text{C}$)：
        $$\text{費用} = \text{體積} \times \text{基本費率}$$

### 3.4 易碎品 (`FragileCargo`)
*   **專屬屬性**：
    *   `packagingType` (字串，例如 `木箱`, `氣泡紙`, `紙箱`)：防護包裝規格。
    *   `maxStackHeight` (整數)：限制最大安全堆疊層數。
*   **倉儲費用公式**：
    *   **堆疊受限補償 (占用地板面積費)**：若 `maxStackHeight <= 2`，加收 $\text{體積} \times 200$；否則加收 $\text{體積} \times 50$。
    *   **包裝處理費**：包裝為「木箱」加收 $100$ 元；「氣泡紙」加收 $50$ 元；其他包裝加收 $30$ 元。
    *   **總費用**：
        $$\text{費用} = (\text{體積} \times \text{基本費率}) + \text{空間占用費} + \text{包裝處理費}$$

## 4. 資料持久化與檔案格式 (Data Persistence)

資料庫採用簡易 CSV 格式存儲於 `data/` 資料夾，以利於 Excel 打開或人工作業。

### 4.1 使用者資料表 (`data/users.txt`)
每行代表一個使用者，以逗號 `,` 分隔：
```text
[角色(Admin/Regular)],[帳號],[密碼]
```
*   **範例**：
    ```csv
    Admin,admin,admin123
    Regular,user,user123
    ```

### 4.2 貨物資料表 (`data/cargo.txt`)
每行代表一項貨物，第一欄為多型類型標籤，藉此還原為衍生類別：
*   **一般貨物**：`General,ID,名稱,重量,體積,基本費率,擁有者`
*   **危險品**：`Dangerous,ID,名稱,重量,體積,基本費率,擁有者,危險等級,UN編號`
*   **易腐品**：`Perishable,ID,名稱,重量,體積,基本費率,擁有者,過期日,溫控要求`
*   **易碎品**：`Fragile,ID,名稱,重量,體積,基本費率,擁有者,包裝方式,最大堆疊層數`
*   **範例**：
    ```csv
    General,C001,普通箱子,100,2,50,user
    Dangerous,C002,鋰電池組,50,0.5,120,user,5,UN3480
    Perishable,C003,新鮮水蜜桃,200,1.2,80,user,2026-07-01,4
    Fragile,C004,高級水晶燈,10,0.8,100,user,木箱,1
    ```

## 5. 非功能性需求 (Non-Functional Requirements)

1.  **記憶體管理安全**：不使用裸指標 (`new` / `delete`)。全部物件使用 `std::shared_ptr` 控制生命週期，確保系統生命週期結束時記憶體能自動安全釋放。
2.  **使用者介面防呆**：
    *   為防止 C++ 於輸入錯誤型態時（如 `int` 欄位輸入中文字）導致 `std::cin` 進入無窮迴圈，實作強制清除緩衝區並重新嘗試的防呆讀取函式。
3.  **色彩人機介面**：
    *   終端機支援標準 ANSI Escapes 轉義碼，包含高亮粗體、青色標題列、黃色警示、紅色警告。
4.  **模組化設計**：
    *   嚴格遵守「一類別一檔案」原則，將宣告 (`.h`) 與實作 (`.cpp`) 徹底分離，並利用標頭檔防護罩 (`#ifndef`) 防止重複 include。
